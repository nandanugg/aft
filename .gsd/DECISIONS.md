| D018 | M001/S02 | arch | Python/Go scope chain strategy | Parent-node walking rather than query captures | tree-sitter queries can't express arbitrary ancestor relationships. Walking parents at extraction time is simple, correct, and fast for the shallow nesting typical in Python and Go. | No |
| D019 | M001/S02 | convention | TypeAlias as 7th SymbolKind | Added TypeAlias beyond the plan's 6 variants | TS type aliases are ubiquitous in agent workflows. | D020 | M001/S02 | convention | Per-language export detection strategy | TS/JS: byte-range containment in export_statement; Rust: pub modifier; Go: uppercase first char; Python: always false | Each language has different export semantics. | D021 | M001/S03 | convention | Command module pattern | `src/commands/` directory with per-command modules (outline.rs, zoom.rs), each exporting a `handle_*` function taking `(&RawRequest, &dyn LanguageProvider) -> Response` | Separates command logic from dispatch. | D022 | M001/S03 | convention | Zoom caller/callee scope | File-scoped only — calls_out and called_by limited to symbols within the same file | Cross-file call graph is M003 (R020). | D024 | M001/S03 | convention | Call extraction algorithm | Recursive node walk with byte-range containment, language-specific node kind matching | Single code path for all 6 languages. call_expression (TS/JS/Go), call (Python), call_expression + macro_invocation (Rust). | D028 | M001/S04 | convention | Checkpoint name collision behavior | Overwrite existing checkpoint on name collision | Agent-friendly — agent doesn't need to check names or delete before creating. The overwrite is implicit and documented in the response. | No |

| D030 | M001/S05 | convention | Symbol edit ranges exclude export keywords | edit_symbol targets tree-sitter's function_declaration node range, not the enclosing export_statement | Tree-sitter scopes the grammar node at the declaration level. | D031 | M001/S05 | convention | validate_syntax returns Option<bool> | None for unsupported languages, Some(true/false) for supported | Clean three-way distinction: valid, invalid, can't-validate. | D032 | M001/S05 | convention | Disambiguation uses success response with code field | Ambiguous symbol/match returns ok:true with a code field, not an error response | Gives the caller structured candidates they can act on. | D033 | M001/S05 | arch | Batch validates before backup | All edits validated against original content before taking backup or modifying file | No backup on validation failure keeps undo history clean — agents don't see phantom backup entries from failed batches.
